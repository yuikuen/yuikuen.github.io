CentOS7部署_Nacos单节点
2022-04-26|springcloudalibaba
Word count: 1.8k|Reading time: 9 min

环境准备

  • 64 bit OS，支持 Linux/Unix/Mac/Windows，推荐选用 Linux/Unix/Mac
  • 64 bit JDK 1.8+；下载 & 配置
  • Maven 3.2.x+；下载 & 配置

安装 JDK

自行下载并上传服务器

安装 Maven

自行下载并上传服务器

修改 Maven Setting.xml 配置文件，配置参考 阿里云云效 Maven 使用指南

安装 MySQL

1）删除旧版本的 MySQL 及相关配置文件

2）安装相关依赖环境并下载源码包(自行下载上传)

3）解压目录并根据需求进行配置安装(基于 cmake 进行配置)

4）数据初始化，创建一个数据库专用账号 mysql(其所属组也为 mysql)

5）拷贝 mysql.server 脚本到 /etc/init.d 目录，编写 MySQL 配置文件，然后启动数据库

6）重置管理员密码并设置安全配置

7）添加服务至开机启动并配置环境变量

安装 Nacos

1）在 GitHub 上下载编译好的 nacos 安装包，下载地址: Release

2）登录 MySQL ，创建数据库并初始化 SQL 数据源

3）配置 nacos 后端数据库，在 nacos 的 conf 目录下修改 application.properties 配置文件

4）脚本默认的启动方式为集群模式，需要修改成 standalone

5）启动服务并测试

6）打开浏览器访问 nacos，默认账密 nacos/nacos

7）配置 systemd 管理 nacos

注：虽然系统已配置了 java 环境，并且直接调用 startup.sh 也能成功启动，但使用服务 service 就无法启动，原因是服务脚本的环境与系统环境变量不能共享
