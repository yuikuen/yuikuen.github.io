K8s问题_Calico网络异常处理
2022-04-07|kubernetes
Word count: 2.1k|Reading time: 12 min

问题一：Calico 服务异常，导致无响应

Warning Unhealthy 2d21h (x2 over 2d21h) kubelet Readiness probe failed: calico/node is not ready: BIRD is not ready: Error querying BIRD: unable to connect to BIRDv4 socket: dial unix /var/run/bird/bird.ctl: connect: no such file or directory

Calico 问题排障，根据提示 估计是没用发现实际真正的网卡

解决方法：调整 calicao 网络插件的网卡发现机制，修改 IP_AUTODETECTION_METHOD 对应的 value 值。官方提供的 yaml 文件中，ip 识别策略（IPDETECTMETHOD）没有配置，即默认为 first-found，这会导致一个网络异常的 ip 作为 nodeIP 被注册，从而影响 node-to-node mesh。我们可以修改成 can-reach 或者 interface 的策略，尝试连接某一个 Ready 的 node 的 IP，以此选择出正确的 IP。
