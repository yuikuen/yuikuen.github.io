019_Jenkins-非流水线项目实现代码审查
2022-04-07|learn
Word count: 431|Reading time: 1 min

以自由风格的 web_demo 项目为例

配置 SonarQube 分析属性，可以指向现有的 Sonar-project.properties 文件或直接在分析属性字段中设置分析属性

1）创建项目，以之前的 maven 构建步骤一样，构建 > 增加步骤 Execute SonarQube Scanner

  • Task to run：SonarQube 执行的命令
  • JDK：Jenkins 全局变量的 JDK 版本
  • Path to project properties：代码审查的配置文件路径（sonar-project.properties）
  • Analysis properties：填写代码审查的配置

根据自身的环境进行填写 Analysis properties

2）点击构建后，可查看构建日志及 SonarQube 项目结果

3）在 SonarQube 点击项目，可详细查看 Bug、异常、问题等具体错误信息
