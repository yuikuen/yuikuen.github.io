023_Jenkins-Docker+SpringCloud微服务构建
2022-04-07|learn
Word count: 2.9k|Reading time: 14 min

项目概述

Jenkins + Docker + SpringCloud 持续集成流程如下：

  • 开发人员把代码提交到 Gitlab 代码仓库
  • Jenkins 从 Gitlab 中拉取项目源码，编译并打成 Jar 包，然后构建成 Docker 镜像，将镜像上传到 Harbor 私有仓库
  • Jenkins 发送 SSH 远程命令，让生产部署服务器到 Harbor 私有仓库拉取镜像到本地，然后创建容器
  • 最后，用户可以访问到容器
服务器名称 IP 地址 安装的软件
代码托管服务器 188.188.3.81 Gitlab
持续集成服务器 188.188.3.82 Jenkins,Maven,Docker
Docker仓库服务器 188.188.3.83 Docker,Harbor
代码审查服务器 188.188.3.84 SonarQube
生产部署服务器 188.188.4.44 Docker,MySQL

注意：上述流程需要提前配置好实验环境，安装好相关软件

项目架构：前后端分离

后端技术栈：SpringBoot+SpringCloud+SpringDataJpa（Spring全家桶）

前端技术栈：NodeJS+VueJS+ElementUI

微服务项目结构：

  • parent：父工程，存放基础配置

  • eureka_server：SpringCloud 的 Eureka 注册中心

  • zuul：SpringCloud 的网关服务

  • common：通用工程，存入工具类

  • gathering：一个简单的业务模块，活动微服务相关逻辑

  • admin_server：基础权限认证中心，负责用户认证（JWT认证）

数据库结构：

  • user：用户认证数据库，存放用户账户数据，对应 admin_server 微服务
  • gathering：活动微服务数据库，对应 gathering 微服务
