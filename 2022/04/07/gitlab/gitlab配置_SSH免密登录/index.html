Gitlab配置_SSH免密登录
2022-04-07|devops
Word count: 761|Reading time: 3 min

概要说明

一般在管理远程主机时，都会使用 SSH 登录，如 ssh user@host ，但每次都需要使用密码；为了方便工作，使用 ssh-keygen 生成的密钥对，然后将公钥添加到目标主机的 ~/.ssh/authorized_keys 文件中，将当前主机纳到可信任主机中，下次就不用输入密码了；

[Gitlab & Github 同理，都支持此种方式的连接]

  • 本机生成密钥对
  • 拷贝公钥到目标主机
  • 测试目标服务是否可用

密钥生成

在~/.ssh下，找到名为id_rsa和id_rsa.pub两个文件。其中id_rsa为私钥，id_rsa.pub为公钥

添加公钥

以 gitlab 为例，找到 头像 -> settings -> SSH Keys，然后将 id_rsa.pub 的内容复制到 key 字段中
