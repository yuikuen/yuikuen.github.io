2022-04-07|helm
Word count: 1.1k|Reading time: 5 min

环境说明

版本信息

  • Kubernetes：v1.20.9
  • Helm：v3.5.4
  • Ingress-Nginx：v3.36.0 & 0.49.0

Ingress-Nginx 官方推荐使用 Helm 进行安装，Helm 安装请自行参考 Kubernetes 部署-Helm

镜像文件：操作过程中，需要提前使用 Github & 阿里云 进行同步镜像

配置修改

1）按照官方说明：添加 ingress-nginx repo 源仓库，再进行 install 就可以使用了，但下述为私有部署环境，需要根据实际进行定制，因此需要下载配置文件进行修改；

2）查看现在版本并进行下载

3）解压并进行修改，修改的地方主要 镜像地址 (在国内可能无法访问国外服务 k8s.gcr.io) 和 更改网络及部署方式为 hostNetwork & DaemonSet

  • 修改 controller & kube-webhook-certgen 镜像地址并关闭 digest 验证
  • 使用宿主机的 80/443 端口，并且使用 Daemonset 固定指定的 Node 作为前端，因使用 deployment 创建 Pod 会偏移
  • 为指定节点添加标签，拥有标签才会部署上去
  • 如使用公有云，有自带的 LoadBalancer 并有分配公有 IP，而私有部署的就采用 CluterIP 就可以
